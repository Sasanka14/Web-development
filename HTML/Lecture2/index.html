Lecture 2: HTML Basics

HTML Timeline

  • HTML 1.0 (1993) - Basic structure for web documents
  • HTML 2.0 (1995) - Standardized core HTML elements
  • HTML 3.2 (1997) - Added tables, applets, and text flow control
  • HTML 4.01 (1999) - Introduced CSS support and better accessibility
  • HTML5 (2014) - Modern web standard with multimedia and APIs

Key Features of HTML

  • Markup language used to structure web pages
  • Uses tags and attributes to define elements
  • Platform-independent and browser-supported
  • Easy to learn and implement
  • Works with CSS and JavaScript for styling and interactivity

HTML vs HTML5

HTML HTML5
No built-in multimedia support Supports audio and video tags
Relies on plugins like Flash No external plugins required
Limited semantic tags Includes semantic tags like header, footer, section
No support for modern APIs Supports APIs like Canvas, Geolocation, and Storage
Less mobile-friendly Designed for mobile and responsive web apps

HTML5 Boilerplate

<!DOCTYPE html> 
<!-- Declares the document type and tells the browser this is HTML5 -->

<html lang="en"> 
<!-- Root element of the HTML document, lang defines the language -->

<head> 
  <!-- Contains metadata and information about the document -->

  <meta charset="UTF-8"> 
  <!-- Sets character encoding to support all languages and symbols -->

  <meta name="viewport" content="width=device-width, initial-scale=1.0"> 
  <!-- Ensures proper scaling and responsiveness on mobile devices -->

  <title>Document</title> 
  <!-- Title shown in the browser tab -->

</head>

<body> 
  <!-- Main content of the webpage goes here -->

</body>

</html>

HTML Tags Overview

Structure Tags (one line each)

  • <html> - Root element that wraps the entire HTML document
  • <head> - Contains metadata and document information
  • <title> - Sets the browser tab title
  • <body> - Holds all visible page content

Semantic Tags (one line each)

  • <header> - Represents the top section of a page or section
  • <nav> - Defines navigation links
  • <main> - Contains the main content of the document
  • <section> - Groups related content together
  • <article> - Represents independent, reusable content
  • <footer> - Represents the bottom section of a page or section

Difference: Structure Tags vs Semantic Tags

Structure Tags Semantic Tags
Define the basic skeleton of the document Define the meaning of the content
Do not describe content purpose Clearly describe content purpose
Mainly used by browsers Used by browsers, search engines, and screen readers
Examples: html, head, body Examples: header, section, article
